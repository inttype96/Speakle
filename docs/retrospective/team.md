## 2. 팀 차원의 회고
### 잘한 점
- 기획과 개발 역할을 명확히 나누고 협업이 원활했음
- Git 브랜치 전략과 코드 리뷰 프로세스를 준수하여 품질 관리에 기여

### 아쉬운 점
- 초반 요구사항 정의가 다소 부족해 중간에 기능 수정이 필요했음
- 일정 관리에서 일부 기능이 예상보다 지연됨

### 배운 점
- 음악/빅데이터/영어학습 도메인을 통합하는 경험을 얻음
- 새로운 기술 스택(Spring Boot 3.x, FastAPI, Weaviate 등) 적용 경험
