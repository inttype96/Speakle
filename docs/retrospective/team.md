## 2. 팀 차원의 회고  

### 잘한 점  
- 기획 단계와 개발 단계를 명확히 구분하고 역할을 분담하여, 전체 프로젝트 진행 과정에서 협업이 원활하게 이루어졌다. 또한 본인의 담당 영역을 넘어서는 부분에서도, 프로젝트 기간 내 완성을 위해 모든 팀원이 적극적으로 기여하였다.  
- AI 코드 리뷰 도구와 팀 내 피드백 과정을 적극적으로 활용하여, 상호 보완적인 협력 체계를 구축함으로써 프로젝트의 완성도를 높일 수 있었다.  

### 아쉬운 점  
- 영어 학습 및 빅데이터 추천이라는 도메인에 대한 이해가 충분하지 않아 초기 요구사항 정의 과정에서 일부 공백이 있었으며, 이로 인해 분석 및 개발 단계에서 추가적인 시간이 소요되고 일부 기능적 변동이 발생하였다.  
- 팀원별 사전 경험과 학습 수준의 차이를 충분히 고려하지 못하여, 기술 스택 선정 과정에서 검증되지 않은 도구나 기술을 도입하게 되었고, 이로 인해 일정 관리와 개발 속도에 일부 영향을 주었다.
- Spotify API 약관(25인 이상 확장 모드 제한) 및 음원 저작권 관련 제약으로 인해, 사용자 개별 계정을 연동하여 서비스를 제공하지 못한 점이 아쉬움으로 남았다.  

### 배운 점
- 생소한 도메인(영어 학습, 빅데이터 추천)이라도 초기 분석과 요구사항 정의 과정을 통해 점차 이해를 넓혀갈 수 있었으며, 도메인 지식의 중요성을 체감할 수 있었다.  
- LLM 및 번역 모델, 추천 시스템과 같은 최신 기술을 실제 서비스 맥락에 적용하면서, 단순 기술 구현이 아닌 **서비스화**를 위한 데이터 전처리·모델 선택·성능 최적화 과정의 필요성을 학습할 수 있었다.  
- 협업 과정에서 팀원 간의 피드백과 코드 리뷰가 프로젝트 완성도를 높이는 데 중요한 역할을 한다는 것을 경험하였다.

## 3. 개인 회고

### 🙋 강성민 (BE)
- **맡은 역할 / 담당 기능**  
  - 회원 및 인증 기능 구현  
    - JWT 기반 인증/인가, Refresh Token 관리  
    - 이메일 인증 및 회원 검증 로직  
  - 학습 콘텐츠 변환(LLM 연동)  
    - LLM 기반 학습 표현(단어·문장·관용구) 추출  
  - 140만 건 가사 번역  
    - 초기 LLM 연동 → 크레딧 과다 소모 문제 해결  
    - GPT-5 의역 데이터(약 40k) 구축 및 LoRA 파인튜닝  
  - 운영 환경 보완  
    - 일자별 로그 파일 저장 기능 도입  
    - 요청 추적 및 로깅 필터 적용  

- **회고**  
  이번 프로젝트는 백엔드 팀원으로서 회원/인증, 학습 콘텐츠 변환(LLM 연동), 140만 건 가사 번역 기능을 담당했습니다.  
  개발 과정에서 가장 큰 제약은 **프로젝트에 지급된 API 호출 크레딧(10만)**이었습니다. 서비스 전반에 걸쳐 임베딩, 키워드 확장, 가사 번역, AI 코드리뷰 등 크레딧 의존도가 높았기 때문에, 전송 포맷 오류 방지, 불필요 문자 제거, 데이터베이스 조회 결과 최소화 등을 통해 **“크레딧 사수”**를 핵심 목표로 두고 개발을 진행했습니다.  

  가사 번역은 처음에는 LLM을 직접 연동했지만, 한 곡당 평균 20크레딧, 스트리밍 시 200크레딧까지 소모되어 실제 서비스 규모에 적합하지 않은 한계가 있었습니다. 이를 해결하기 위해 사전 학습된 모델을 탐색한 뒤 en→ko 파인튜닝 모델을 선정했고, 별도로 GPT-5를 활용하여 약 4만 건의 의역 데이터를 구축해 LoRA 방식으로 추가 파인튜닝을 수행했습니다.  

  진행 과정에서 기간적 압박이 컸고, 팀에서는 “일단 연동을 우선해야 한다”는 요구가 있었던 반면, 저는 “효율적인 방식으로 개선해야 한다”는 고민이 있었습니다. 그러나 결과적으로는 크레딧 절약과 서비스 품질 모두를 고려한 방안을 마련할 수 있었고, 이를 통해 의미 있는 경험을 얻을 수 있었습니다.
